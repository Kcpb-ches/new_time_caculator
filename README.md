# 解送人犯報告書系統

基隆市警察局刑事警察大隊科技偵查隊　v3.0

本系統提供網頁介面，協助警察人員快速填寫解送人犯報告書，並一鍵匯出符合格式的 Word 文件（.docx）。

---

## 功能說明

### 一、案件基本資訊

| 欄位 | 說明 |
|------|------|
| 警察機關 | 下拉選單，選擇所屬警察局 |
| 所屬分局 | 依警察機關連動顯示分局清單，選擇後自動帶入機關地址 |
| 單位（派出所/隊） | 手動輸入，出現於報告書標題（如「○○派出所解送人犯報告書」） |
| 機關地址 | 選擇分局後自動填入，可手動修改 |
| 是否為通緝犯 | 影響法定解送期限：一般為 **16 小時**，通緝犯為 **24 小時** |
| 發文日期 | 預設帶入當日，格式為民國年 |
| 發文字號 | 手動輸入 |
| 致送地方檢察署 | 預設帶入「臺灣基隆地方檢察署」，可修改 |

---

### 二、犯罪嫌疑人／被移送法人

點擊「**＋ 新增犯嫌**」可新增多名移送對象，每個對象可獨立展開/收合。

#### 移送對象類型

每張卡片頂端可選擇：

- **自然人**（預設）
- **法人**

##### 自然人欄位

| 欄位 | 說明 |
|------|------|
| 姓名 | |
| 性別 | 男 / 女 |
| 出生年月日 | 民國年格式（年/月/日） |
| 身分證號 | |
| 戶籍地址 | |
| 現住地址 | 可勾選「同戶籍」自動帶入 |
| 選任辯護人 | 預設「無」 |
| 拘提時間 | 預設帶入前一日 00:00，格式為民國年 |
| 拘提地點 | |

##### 法人欄位

| 欄位 | 說明 |
|------|------|
| 公司名稱 | |
| 統一編號 | |
| 代表人姓名 | |
| 公司所在地 | |
| 拘提時間 | |
| 拘提地點 | |

#### 智慧照片 OCR 匯入（限自然人）

點擊「**📷 智慧照片匯入**」，上傳警察系統「智慧照片」截圖，AI 自動辨識並填入以下欄位：
- 姓名、性別、身分證號、出生年月日、戶籍地址、現住地址

> **注意：** OCR 每個 IP 每日限使用 **20 次**，辨識結果請人工確認後再匯出。

#### 法定障礙事由

每位移送對象皆有獨立的「**法定障礙事由**」頁籤，支援以下 10 項事由：

| 編號 | 事由 |
|------|------|
| 1 | 因交通障礙或其他不可抗力之事由所生不得已之遲滯 |
| 2 | 在途解送期間 |
| 3 | 依第一百條之三第一項規定不得為詢問者（夜間詢問禁止） |
| 4 | 因被告或犯罪嫌疑人身體健康突發之事由，事實上無法詢問者 |
| 5 | 被告或犯罪嫌疑人表示已選任辯護人，或強制辯護案件，因等候辯護人到場致未予訊問者 |
| 6 | 依第三十五條第三項通知陪同在場之人到場致未予訊問者 |
| 7 | 被告或犯罪嫌疑人不通國語須由通譯傳譯，因等候通譯到場致未予訊問者 |
| 8 | 經檢察官命具保或責付之被告，在候保或候責付中者 |
| 9 | 犯罪嫌疑人經法院提審之期間 |
| 10 | 依第三十四條第二項辯護人與偵查中受拘提或逮捕之被告或犯罪嫌疑人接見之時間 |

- 勾選後可設定障礙事由的**起訖時間**，系統自動計算扣除分鐘數
- 事由 3（夜間詢問禁止）勾選後，可點擊「**自動帶入日沒/日出**」，依據拘提地縣市自動查詢當日日沒與日出時間（中央氣象署 API）

### 三、案情說明

| 欄位 | 說明 |
|------|------|
| 犯罪嫌疑事實 | 自由填寫案情描述 |
| 所犯法條 | 適用法條條文 |

---

### 四、所附證據資料

勾選隨案移送的證據資料（可複選）：

- 嫌疑人、被害人及相關證人之筆錄
- 扣案證物：詳如扣押目錄表
- 嫌疑人身分證件、指紋卡、口卡片
- 嫌疑人刑案資料
- 逮捕通知書（須記載逮捕日、時、分）
- 搜索票及搜索扣押筆錄、收據

---

### 五、聲請羈押事由

勾選聲請羈押相關事由（可複選）：

- 嫌疑人係通緝犯
- 有同案共犯未到案
- 有同案共犯已移送偵辦
- 有重要證物未起獲或有重要證人待詢問
- 嫌疑人有繼續加害被害人之危險
- 為逮捕拘提時有無逃脫行為
- 其他（可填入自訂文字）

---

### 六、解送期限摘要

頁面下方自動顯示每位移送對象的期限摘要表，包含：

- **拘提時間**
- **法定障礙事由扣除時間**（合計）
- **解送期限**（依法定期限計算）

期限顏色提示：
- 綠色：時間充裕
- 黃色：距期限不足 1 小時（緊急）
- 紅色：已逾期

---

### 七、匯出報告書

點擊右上角「**匯出報告書 (.docx)**」按鈕，下載 Word 格式報告書。

匯出格式說明：
- 標題：`[單位]解送人犯報告書`，字體標楷體 **26pt**
- 內文：標楷體 **14pt**
- 第二列：發文日期、發文字號同列顯示
- 移送對象：自然人顯示「犯罪嫌疑人（被告）」，法人顯示「被移送法人」
- 最後一列：解交人與接收人欄位各佔 50% 寬度
- 法定障礙事由：僅列出已勾選並填入起訖時間的項目，並顯示各人合計扣除時間

## 技術架構

| 項目 | 技術 |
|------|------|
| 前端 | React 18（CDN UMD）、Babel Standalone、Tailwind CSS |
| Word 匯出 | docx@7（CDN UMD，`window.docx`） |
| OCR 後端 | Vercel Serverless Function（`api/ocr.js`） |
| AI 辨識 | Anthropic Claude API（claude-sonnet-4-6） |
| 流量限制 | Upstash Redis（IP 每日 20 次） |
| 日出日沒 | 中央氣象署開放資料平台 API |
| 部署 | Vercel |

---

## 環境變數設定（Vercel）

| 變數名稱 | 說明 |
|----------|------|
| `ANTHROPIC_API_KEY` | Anthropic API 金鑰（必填） |
| `KV_REST_API_URL` | Upstash Redis REST URL（連接 Upstash 後自動設定） |
| `KV_REST_API_TOKEN` | Upstash Redis Token（連接 Upstash 後自動設定） |

---

## 注意事項

- 本系統僅供**基隆市警察局刑事警察大隊科技偵查隊**內部使用
- OCR 辨識結果可能因截圖解析度不足而有誤差，匯出前請務必人工核對所有欄位
- 系統不儲存任何個人資料，所有資料僅存於瀏覽器工作階段（重新整理後清空）
